HTML5, touch_event, zepto

移动设备触摸屏事件

zepto在移动设备上得到大范围的使用，一方面是因为其体积很小，加载速度快，另外一方面就是由于它对触摸屏事件的支持很优秀。它有一个touch模块，用来处理移动设备上的点击（tap）和滑动（swipe）事件，主要就是通过对touchEvent的属性来实现的。

属性列表

Touch对象有如下几个属性：

  • identifier
  • target
  • screenX
  • screenY
  • clientX
  • clientY
  • pageX
  • pageY

identifier是每个触摸点的身份证明，以此来区分不同的触摸点。target是触摸点所在的元素。其他的属性通过名称就都可以知道了。

TouchList对象包含一个触摸事件中的所有触摸点的列表。有一个length属性和item方法，item(0)可以获得列表中第一个触摸点。

TouchEvent对象有如下几个属性：

  • touches
  • targetTouches
  • changedTouches
  • altKey
  • metaKey
  • ctrlKey
  • shiftKey

touches是与屏幕接触的所有触摸点数组，targetTouches是目标DOM元素上的触摸点数组，changedTouches是发生改变了的触摸点数组，它们都是touchList对象，后面四个都是布尔值，表示触摸时相应的键是否被按下。

事件类型

touchEvent有以下四种事件类型：

  • touchstart //开始
  • touchend //结束
  • touchmove //移动
  • touchcancel //取消

其他

触发touchend事件时，e.targetTouches.length已经为0，如果要获得最后点的信息，应该通过e.changedTouches[0]来获得。

禁止缩放和滚动：
禁止缩放使用viewport即可，禁止滚动需要禁止touchmove的默认事件：

document.body.addEventListener('touchmove', function(event) {
  event.preventDefault();
}, false);

HTML5, notification

使用HTML5 Notification API开启浏览器桌面提醒

前两天发现一款app，安装了app和相应的chrome插件以后，可以将安卓手机上的消息在电脑桌面上弹窗显示。这样一来，工作的时候就可以完全把手机扔在一边安心工作了，而不用时时查看是否错过了什么消息，如下图所示：

这个应用实现的原理其实很简单，大概原理是，监听手机的消息系统，当有新消息的时候就同步到服务器，服务器接收到通知，然后通过HTML5的Notification API推送消息到电脑。之前有玩过这个API，这次再完整地学习一遍，记录下来。

如何使用

申请权限

出于安全考虑，要发送桌面消息，需要先申请用户授权。Notification对象提供了一个静态的方法——requestPermission()，它接收一个回调函数作为参数，并把返回值传递给回调函数作为参数：

Notification.requestPermission(function(status){
            if(Notification.permission !== status){
                Notification.permission = status;
            }
        });

返回值为字符串，有以下三个值：

  • default
  • granted
  • denied

默认为default，也就是需要询问，表现和denied一样。

数组合并排序的实现

美团网前端面试题：

已知两个有序的数组，将它们合并成也是有序的一个数组。举例如下：
输入：A = [1, 2, 4, 7], B = [3, 5, 6, 8]，输出 C = [1, 2, 3, 4, 5, 6, 7, 8]。

1.简单粗暴的第一个想法，直接使用数组的concat()方法把两个数组拼接成为一个数组，然后再调用sort()方法来进行排序，代码实现如下：

     function concatSort(a,b){
         var arr = a.concat(b).sort(function(x,y){
            return x-y;
        })
        return arr;
    }

2.array方法的巧妙运用，每次都把两个数组第一个数字中的较小者放到结果数组后面，同时删除较小数字所在数组中的这个数字，再进行下一轮比较，直到其中一个数组空了为止，然后把剩余部分拼接到结果数组的后面。
